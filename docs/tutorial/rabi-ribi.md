# **Rabi-Ribi联机教程**
:::tip
建议在PC端阅读本文以获得最佳体验，同时教程默认在PC上进行。
:::
众所周知，菟最大的联机阻碍往往是：`根本不知道有联机功能`，不知有联机功能很大程度上阻碍了菟联机的传播。<br><br>然而就算知道有联机功能后，往往又会被联机的折腾劝退，毕竟现在菟社区联机以口口相传为主，确实缺少一些实用工具和一个完成度较高的教程。同时在国内复杂的跨网环境下。你游还有着最容易翻车的联机形式：`远程桌面`。<br><br>正是上述原因催生了这篇教程，话不多说，我们将从一般的联机流程开始了解，并对大概率会遇见的问题提供解决思路，助力各位艾莉娜（或是其实你喜欢猫咪?）在拉比拉比岛迎接激爽的战斗体验！
<br>
<br>
<br>
一般来说，联机的最短路径是这样的：

1. 至少有一份Rabi-Ribi游戏。
2. 抓到至少一位`朋友`（这太难了）。
3. 选择一个合适的串流软件。
4. 开始享受（受苦）。

接下来就从`串流软件`开始介绍，打通联机的路径。

## 串流软件选择
:::code-group
```bash [UU远程]
优点：
    流程最简单，注册登录后选择远程协助搞定；
    若直连不通，有免费的中转服务器兜底。

缺点：
    无法调整编解码器，对应延迟很高，对于控制端而言总延迟比较影响操作；
    同带宽下画质较差；
    仅可双人联机；
    不支持模拟手柄（P2必须手柄，客机若是键盘玩家等于没法玩，只能对调位置）。

适用：
    1. 管你这那的，先玩起来再说。
    2. 带新。
    3. 难以解决网络问题。
    3. 非深入/长期游玩（尤其是高难度）。
```
```bash [Parsec]
优点：
    PC连PC可以说是目前编解码延迟最低的串流软件，调度很好；
    可以选择只串流特定应用。
    自带连麦回声消除（主Discord）。

缺点：
    国外软件，注册或连接容易被阻断，需要优化（好在不难）；
    容易忘记密码（？）。
    
适用：
    1. 追求最低延迟。
    2. 长期游玩或打高难度。
    3. 受控方有显著的隐私需求。
    4. 受不了连麦回音。     
```
```bash [Sunshine+Moonlight]
优点：
    可自定义项巨大多，能根据自己配置深度定制，玩得转的可能会比较喜欢这一点。

缺点：
    纯血的点对点连接，必须依赖直接可达的网络路径，不具备NAT穿透（打洞），
    因此只可通过物理内网或公网的IP或自建虚拟局域网实现连接，
    对使用者网络知识能力有要求。

适用：
    1. 局域网多设备联机（对玩菟来说，科幻片）。
    2. 拥有线路带宽都尚可的内网穿透或虚拟局域网。
```
```yaml [Steam Link/远程同乐/其他]
总体评价：可自定义项过少/体验不佳/难以稳定/过于冷门，不推荐。
:::

## 联机模式讲解
### `如何进入联机模式`

在游戏内的**存档点/回血点/BossRush开场前五秒**按`F1`召唤客机即可开始游玩，最多`四人`同屏。

### `如何操控`

多人模式下，主机P1被锁定为键盘操控，P2/3/4锁定为手柄操控且共用一套游戏手柄键位。
<br><br>`客机键盘玩家`需要单独下载`Keyboard Splitter`将按键映射为一个模拟手柄，而客机手柄玩家连上就能玩。<br>
:::details 关于Keyboard Splitter
[下载地址](https://wwazt.lanzoum.com/iuTyx3nji9ib)，需要安装虚拟手柄驱动重启电脑。<br>
<br>进入软件后`插槽数`选择为`1`，参照主机的手柄键位，可以在较熟悉手柄的朋友或AI的帮助下修改映射关系，之后点击`开始`即可。<br><br>另外测试时请确认虚拟手柄选择了正确的键盘。<br>
<br>![图 7](https://img.flametn7.cc.cd/2026/04/18/10c0d2fe9614853ebf0621c5ed12dbbd476c0a80e12d6c347c73c536af96a780%24%7Bext%7D.png)  
:::
:::warning

串流识别手柄有个小坑，请等待所有人都连接上主机并且各自按过按键让主机识别了手柄再打开游戏，否则客机`无法`操控，可以启动前通过`Steam输入`查看手柄连接状态。

客机若在游戏中途断开，重连后一般`不需要`重启游戏也可以继续操作。主机使用键盘时最好拔掉自己的手柄以防干扰客机。

:::

在大于2人（尤其是有手柄玩家）的情况下，最好统一游戏内键位并让不同玩家按照统一键位自行修改按键映射，做主机的人如果自身在单机时也是手柄玩家，可以用`Steam输入`做到快速的按键方案切换。

此外，上述主客机是以`远程桌面`的控制与被控来区分的，游戏内主机客机没有严格的区分，能接管键盘控制P1就可以做主机，只不过控制端玩P1需要承受延迟。

## 常见问题

### 为什么连不上
以`Parsec`为例，以下几个方面是可以排查的方向或解决思路，从左到右大抵是`越来越折腾`的排序：
:::code-group
```bash [Parsec报错]
错误代码：14003（笔记本混合显卡问题）
    受控端需要调整设置↓
    Windows11→显示设置→高级显示设置→显示卡→手动添加Parsec主程序强制核显运行。
    Windows10→显示设置→图形设置→手动添加Parsec主程序并强制核显运行

错误代码：6023（网络环境问题）
    如果反复出现此报错说明P2P直连失败了，它通常会伴随一个二级代码可供排查。
    -11000、-11004或-11010： 见“IPv6”与“Nat类型”两栏。
    -11011：把Parsec软件自身的 UPnP设置打开，位于Network选项卡。
```

```yaml [防火墙]
确保系统或第三方软件防火墙没有阻拦串流，注意放行。
```

```bash [换位]
试试主客机位置对调，或找网络环境好的人当主机。
```

```bash [换软件]
折腾不动？UU远程有直接可用免费中转服务器。
```

```bash [IPv6]
要是前几项还没有搞定，那么现在需要自行改善网络环境了。
在折腾Nat类型之前，现在的家庭宽带默认分配公网IPv6地址，请确认：
    1.电脑网卡是否向网关申请了IPv6，Windows网络适配器的IPv6有可能要自行开启；
    2.路由器是否支持并开启IPv6；
    3.（非桥接）光猫若无IPv6，联系装维或运营商开启；
    4.（非桥接）光猫可能要检查IPv6防火墙。

与IPv4相比，公网IPV6联机需要双方都具备公网IPV6地址
检测自己有没有公网IPV6---→ https://www.test-ipv6.com/

值得一提的是，国内移动网络几乎都支持IPv6，如果你流量上网足够稳定，usb共享给电脑，
以此获得一个公网IPv6也是可行的省事办法，但是请注意调整串流带宽防止流量消耗过大。
```

```yaml [Nat类型]

若确定无法获取任何公网IP，联机的至少其中一方需要改善Nat类型，

    1.联系装维或运营商客服远程改好桥接；
    2.如果第一步是“竟然不许”的结果，并且要不到超密，可以用非常规手段查询超密。

    4.开始享受吧！

补充：如果没有路由器只有光猫就不用理会桥接，获取超密开启UPnP即可。
```

## 联机体验优化补丁

想必你已经发现联机机制（或者说Bug）对客机十分不友好，还会让游戏难度大幅度提升，好在我们跑遍Github发现了一个[补丁](https://github.com/Sinamer/BunnyGamePatch)！
